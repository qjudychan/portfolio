Designing a point of sale system using the Carbon stablecoin network

The following reflects a point of sale system that offers two alternative mobile-based methods for the buyer to select from, eliminating the need for the merchant to have chip readers, swipers, scanners, etc.

Buyers can select from:

  • Option A - Engage with Custom QR Code:

    1. The system will generate a unique QR code for the transaction



    2. Buyer scans the code using their smartphone, either through a Carbon native app or through their camera app, if possible (e.g. on iOS 11 and later), which can link them directly to the native app




  • Option B - Numerical Two-Step Verification:

    This option accomodates those without a smartphone, those with a smartphone but no data plan, or those with a smartphone that has a broken camera

    1. Buyer enters their phone number so that the system can send a custom generated numerical code for the transaction



    2. A code is texted to the phone number entered; the text message will also provide a link that the user can tap on to complete the transaction in the Carbon app

    3. Buyer inputs the code that was texted to them; alternatively, the user could tap the link in the text to avoid having to manually type the code, which would make the transaction less error prone and more efficient
